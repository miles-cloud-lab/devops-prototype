# 流水线构建版本追溯 PRD

## 1. 背景

用户排查构建、部署或回滚问题时，需要快速确认本次流水线运行对应的分支、Commit 和提交说明。

## 2. 交互说明

### 2.1 代码源卡片

在“最近运行”页的代码源卡片中增加“版本详情”入口。

单代码源和多分支模式均使用同一个入口名称：`版本详情`。

单分支模式下，仅支持选择一个引用对象，且只能二选一：一个分支或一个标签。

多分支模式下，支持选择多个分支、多个标签，或分支与标签混选。代码源卡片不再直接展示运行引用，仅保留发布分支；运行引用统一放到弹窗中展示。

### 2.2 版本详情弹窗

点击“版本详情”后打开“版本信息”弹窗。

弹窗顶部展示本次运行的发布分支。

弹窗内展示运行引用表格，字段包括：

| 字段 | 说明 |
| --- | --- |
| 引用名称 | 本次运行涉及的分支或标签 |
| 引用类型 | 当前引用是分支还是标签 |
| commitId | 引用对应的 Commit，默认展示短 Commit，hover 展示完整 Commit |
| 提交说明 | 引用对应的提交说明 |

### 2.3 单代码源与多分支差异

单代码源：一次仅允许选择一个分支或一个标签，发布分支与本次实际运行引用一致，运行引用表格中展示该引用及其 Commit、提交说明。

多分支：发布分支展示本次集成/发布使用的分支，运行引用表格中展示参与集成的运行分支/标签及其 Commit、提交说明。
