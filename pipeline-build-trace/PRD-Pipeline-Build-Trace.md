# 流水线构建版本追溯 PRD

## 1. 背景

当前 Flux DevOps 流水线名称是创建流水线时的固定资产名称，不会随着每次运行动态变化。用户在排查构建失败、部署异常、版本回滚或环境差异时，更关心的是“本次运行到底基于哪份代码构建”，而不是流水线资产本身叫什么。

现有页面对本次运行的代码分支、Commit、提交说明、多分支集成关系展示不足，导致用户需要在日志、代码仓库、部署记录之间来回核对。

本需求的本质是增强流水线运行实例的代码版本追溯能力。

## 2. 目标

1. 在“最近运行”的代码源卡片中展示本次运行的核心代码版本快照。
2. 支持单分支模式和多分支模式的差异化展示。
3. 多分支模式下，参考云效的卡片与弹窗表达，提供“分支详情”入口，用弹窗展示完整运行分支列表。
4. 运行历史和部署历史可追溯到本次运行对应的代码版本。
5. 流水线名称仍保持固定资产名称，不因每次运行动态变化。

## 3. 竞品参考

参考云效“最近运行”页中的代码源节点卡片与分支信息弹窗的信息组织方式，不参考其整体导航、布局、颜色和页面框架。原型和最终实现仍保持 Panther / Flux DevOps 现有页面风格，仅优化代码源卡片内的字段呈现。

### 3.1 单分支模式

云效卡片展示信息：

| 信息 | 说明 |
| --- | --- |
| 代码仓库 | 展示仓库名称，过长时省略 |
| 运行分支 | 展示本次触发或选择的分支，如 `master` |
| Commit | 展示短 Commit，如 `f29435fe` |
| 提交说明 | 展示一行提交信息，过长省略 |

单分支模式不需要弹窗，卡片内直接展示足够定位本次运行代码的信息。

### 3.2 多分支模式

云效卡片展示信息：

| 信息 | 说明 |
| --- | --- |
| 代码仓库 | 展示仓库名称，过长时省略 |
| 发布/集成分支 | 展示系统创建或使用的发布分支，如 `release_4902259_2` |
| 构建 Commit | 展示本次最终用于构建的短 Commit |
| 分支详情 | 链接入口，点击后打开弹窗 |

分支详情弹窗展示信息：

| 信息 | 说明 |
| --- | --- |
| 发布分支 | 弹窗顶部展示本次发布/集成分支 |
| 运行分支 | 表格展示参与本次运行的源分支 |
| commitId | 每个运行分支对应的 Commit，可展示短值或截断长值 |
| 提交说明 | 每个运行分支对应的提交说明 |

## 4. 需求范围

### 4.1 本期包含

1. 最近运行页代码源卡片展示代码仓库、分支、Commit、提交说明。
2. 单分支模式：卡片内展示运行分支、短 Commit、提交说明。
3. 多分支模式：卡片内展示发布/集成分支、构建 Commit，并提供“分支详情”入口。
4. 分支详情弹窗展示发布分支和完整运行分支列表。
5. 运行历史列表补充代码版本信息，至少包含分支和短 Commit。
6. 部署历史复用已有“代码版本”字段，支持点击查看代码版本详情。
7. 流水线运行开始时固化代码版本快照，保证临时分支删除后仍可追溯。

### 4.2 本期不包含

1. 不修改流水线名称生成规则。
2. 不支持用户自定义运行记录标题模板。
3. 不做自动回滚。
4. 不做跨仓库复杂依赖关系可视化。
5. 不负责恢复或保留已经删除的临时分支，只保留运行快照。

## 5. 页面设计

### 5.1 最近运行页代码源卡片

入口：

```text
流水线详情 > 最近运行 > 代码源节点
```

卡片设计原则：

1. 页面整体保持 Panther 现有风格：顶部深色导航、左侧深色菜单、浅蓝灰画布、阶段泳道、节点连线均不调整。
2. 不改变代码源卡片原有样式、位置、节点状态、耗时和日志入口。
3. 卡片顶部仍展示代码仓库图标和仓库地址/仓库名。
4. 原有项目、分支信息保留，在其后补充 Commit 和提交说明。
5. 原卡片中的仓库、项目、分支、日志等图标保留，不替换成文字标签。
6. 单代码源和多分支/多代码源保持一致的交互样式：卡片保留原有代码源摘要，通过详情入口查看 Commit 和提交说明。
7. 多分支/多代码源模式继续展示原有代码源摘要和分支详情入口，卡片不展示 Commit 和提交说明。
7. 卡片字段超长时单行省略。
8. 卡片点击行为保持现有节点逻辑，不因本需求新增右侧抽屉。

单分支卡片展示：

```text
Codeup-Demo
master
f29435fe
更新 README.md co...
```

多分支卡片展示：

```text
Codeup-Demo
release_4902259_2
aacb9ff4
分支详情
```

### 5.2 分支详情弹窗

入口：

```text
多分支模式代码源卡片 > 分支详情
```

弹窗内容：

1. 标题：分支信息。
2. 单分支和多分支弹窗顶部统一展示构建分支：单分支为本次选择的分支，如 `feature-20260515-saas`；多分支为本次集成的基准分支，如 `master`。
3. 分支明细表格：展示本次参与构建或集成的分支列表。
4. 表格列统一为：分支、commitId、提交说明；commitId 默认展示短 Commit，hover 展示完整 Commit。
5. 每个运行分支可展示该分支自己的 commitId 和提交说明；这部分不直接铺在卡片上。
6. 弹窗只展示运行时固化的快照，不依赖临时分支当前是否还存在。

### 5.3 运行历史列表

入口：

```text
流水线详情 > 运行历史
```

补充字段：

| 字段 | 说明 |
| --- | --- |
| 运行编号 | 本次运行编号 |
| 状态 | 成功、失败、运行中、终止 |
| 代码版本 | 单分支展示 `master_f29435fe`；多分支展示发布分支和构建 Commit |
| 分支 | 单分支展示运行分支；多分支展示“多分支（N）” |
| 提交说明 | 展示一行提交说明，超长省略 |
| 触发人 | 本次运行触发人 |
| 开始时间 | 本次运行开始时间 |

### 5.4 部署历史列表

入口：

```text
流水线详情 > 部署历史
```

现有部署历史已有“代码版本”字段，本期不新增重复字段。优化规则：

1. 代码版本支持点击。
2. 点击后展示代码版本详情。
3. 有运行快照时展示完整 Commit、分支、提交说明。
4. 历史记录没有快照时，仅展示已有代码版本文本，不报错。

## 6. 数据固化规则

流水线运行开始时，需要固化本次运行的代码版本快照。

建议字段：

| 字段 | 说明 |
| --- | --- |
| pipelineRunId | 流水线运行 ID |
| pipelineName | 固定流水线名称 |
| runNumber | 运行编号 |
| triggerMode | 手动、代码提交、定时等 |
| triggerUser | 触发人 |
| sourceRepo | 代码仓库 |
| branchMode | 单分支 / 多分支 |
| sourceBranch | 单分支运行分支，或多分支源分支列表 |
| publishBranch | 多分支发布/集成分支 |
| baseBranch | 多分支基础分支 |
| commitSha | 本次最终构建 Commit |
| shortCommitId | 短 Commit |
| commitMessage | 提交说明 |
| commitAuthor | 提交人 |
| commitTime | 提交时间 |
| branchIntegrations | 多分支运行分支列表，包含分支、commitId、提交说明 |

说明：

1. 页面展示依赖运行快照，不依赖运行后仍能查到临时分支。
2. 多分支临时分支删除后，仍可通过 `commitSha` 追溯本次构建代码。
3. 源分支运行后被删除时，快照仍保留当时的分支名、Commit 和提交说明。

## 7. 展示规则

### 7.1 Commit

1. 列表和卡片默认展示短 Commit。
2. 详情和弹窗中可展示完整 Commit，过长时中间省略。
3. 支持复制 Commit。
4. 如果代码仓库支持跳转，Commit 可跳转提交详情。

### 7.2 提交说明

1. 卡片中展示一行，超长省略。
2. 详情中展示完整提交说明。
3. 无提交说明时展示 `--`。

### 7.3 多分支

1. 卡片保持原有比例，不展开每个源分支的 Commit 和提交说明。
2. 单代码源卡片不直接展示 Commit 和提交说明，仅提供分支详情入口。
3. 多分支/多代码源卡片不展示 Commit 和提交说明，仅提供分支详情入口。
4. 弹窗展示完整运行分支列表，以及每个分支自己的 Commit 和提交说明。
5. 运行分支较多时，弹窗表格支持滚动。

## 8. 异常场景

| 场景 | 展示规则 |
| --- | --- |
| 无法获取 Commit | Commit 展示 `--`，详情提示“未获取到代码提交信息” |
| 临时发布分支已删除 | 发布分支仍按快照展示，不影响 Commit 展示 |
| 源分支已删除 | 源分支名称正常展示，状态可标记“已删除” |
| 多代码源 | 本期先展示主代码源，多代码源完整展示作为后续能力 |
| Commit 跳转失败 | 保留复制能力 |
| 历史数据无快照 | 展示已有文本或 `--`，不报错 |

## 9. 验收标准

1. 最近运行页单分支代码源卡片可展示仓库、运行分支、短 Commit、提交说明。
2. 最近运行页多分支代码源卡片可展示仓库、发布/集成分支、构建 Commit 和“分支详情”入口。
3. 点击“分支详情”可打开弹窗，弹窗展示构建分支、分支明细、commitId 和提交说明。
4. 弹窗关闭按钮和右上角关闭图标均可关闭弹窗。
5. 多分支临时分支删除后，运行详情仍能展示本次构建 Commit。
6. 运行历史列表可展示代码版本、分支、短 Commit、提交说明。
7. 部署历史已有“代码版本”列保持不变，并支持点击查看详情。
8. 历史无快照数据时页面不报错，展示 `--`。
9. 流水线名称不随每次运行变化。
10. 原型图中的卡片和弹窗样式与云效参考图的信息层级保持一致。
