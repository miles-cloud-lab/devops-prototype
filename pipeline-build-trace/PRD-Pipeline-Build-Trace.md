# 流水线构建版本追溯 PRD

## 1. 背景

用户排查构建、部署或回滚问题时，需要快速确认本次流水线运行对应的分支、Commit 和提交说明。

## 2. 交互说明

### 2.1 代码源卡片

在“最近运行”页的代码源卡片中增加“分支详情”入口。

单代码源和多分支模式均使用同一个入口名称：`分支详情`。

多分支模式下，代码源卡片不再直接展示运行分支，仅保留发布分支；运行分支统一放到弹窗中展示。

### 2.2 分支详情弹窗

点击“分支详情”后打开“分支信息”弹窗。

弹窗顶部展示本次运行的发布分支。

弹窗内展示运行分支表格，字段包括：

| 字段 | 说明 |
| --- | --- |
| 分支 | 本次运行涉及的分支 |
| commitId | 分支对应的 Commit，默认展示短 Commit，hover 展示完整 Commit |
| 提交说明 | 分支对应的提交说明 |

### 2.3 单代码源与多分支差异

单代码源：发布分支与本次实际运行分支一致，运行分支表格中展示该分支及其 Commit、提交说明。

多分支：发布分支展示本次集成/发布使用的分支，运行分支表格中展示参与集成的运行分支及其 Commit、提交说明。
