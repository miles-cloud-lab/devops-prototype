# AI 代码评审功能 PRD

## 1. 需求背景

当前 Flux DevOps 已支持在流水线中配置「代码评审」任务。流水线执行到该任务后，会根据源分支与目标分支生成代码对比，等待人工评审人完成审批后继续执行。

现有评审能力主要依赖人工阅读 diff，存在以下问题：

1. 评审人需要逐行阅读变更，面对多文件、多提交场景时定位风险效率较低。
2. 评审结论依赖个人经验，代码质量、安全风险、测试遗漏等问题难以形成稳定检查口径。
3. 流水线只有人工审核状态，缺少自动初审结果，无法在列表页快速识别高风险评审单。
4. 评审过程缺少结构化问题沉淀，后续无法统计误报率、采纳率和项目风险趋势。

业内云上 DevOps 平台通常将 AI 代码评审作为 PR/MR 的自动初审能力，结合 diff、相关上下文和团队规则，在人工审批前生成摘要、行内评论、风险等级和测试建议。AI 不替代人工审批，而是帮助评审人更快定位风险。

## 2. 需求目标

1. 在现有代码评审任务中增加 AI 辅助评审能力。
2. AI 基于源分支与目标分支 diff 自动生成评审摘要、风险问题、行内建议和测试建议。
3. 在评审列表页展示 AI 评审状态与风险等级，帮助评审人优先处理高风险评审单。
4. 在 diff 详情页展示 AI 总览、文件风险标识、行内评论和右侧 AI 评审抽屉。
5. 支持在流水线代码评审任务中配置是否启用 AI 辅助评审。
6. 保留人工审批作为最终评审结论，AI 仅作为辅助判断，不改变现有审批规则。

## 3. 产品定位

功能名称：AI 辅助评审

功能定位：代码评审任务中的自动初审能力。

不建议命名为「AI 审批」或「AI 自动通过」，避免用户误解 AI 具有最终审批权。

## 4. 用户角色

| 角色 | 诉求 |
| --- | --- |
| 流水线配置人 | 在代码评审任务中启用或停用 AI 辅助评审 |
| 代码提交人 | 在人工评审前看到 AI 问题，提前修复明显风险 |
| 评审人 | 快速了解变更摘要、风险点和重点文件，提高评审效率 |
| 管理员 | 配置团队规则、模型策略、是否允许豁免阻断 |

## 5. 需求范围

### 5.1 本期包含

1. 流水线「代码评审」任务新增 AI 辅助评审配置。
2. 代码评审列表新增 AI 状态、风险等级、问题数、AI 摘要。
3. 代码评审详情页新增 AI 总览、右侧抽屉、文件风险标识、行内 AI 评论。
4. 支持手动重新评审。
5. 支持按严重、高、中、低展示问题等级。
6. 支持 AI 评论反馈：采纳、忽略、误报、转人工确认。
7. AI 结果仅提示，不作为本期流水线阻断条件。

### 5.2 本期不包含

1. 不支持 AI 自动合并代码。
2. 不支持 AI 直接代替人工审批人。
3. 不支持跨项目自动学习私有代码规范。
4. 不支持复杂多轮代码修复 Agent。
5. 不支持全仓深度重构建议。
6. 不支持在流水线任务中配置触发方式、评审范围、评审规则、阻断策略。

## 6. 业务流程

### 6.1 自动评审流程

1. 流水线执行到代码评审任务。
2. 系统根据任务中的源分支、目标分支生成 diff。
3. 如果任务启用 AI 辅助评审，则创建 AI 评审任务。
4. AI 获取 diff、文件路径、提交信息和必要上下文。
5. AI 生成评审摘要、风险问题、行内评论、测试建议。
6. 页面展示 AI 评审结果。
7. 人工评审人结合 AI 结果完成通过或驳回。
8. 人工审批规则保持不变，AI 结果不影响本期流水线通过逻辑。

### 6.2 手动重新评审流程

1. 用户在评审详情页点击「重新评审」。
2. 系统检查源分支与目标分支最新 diff。
3. 重新创建 AI 评审任务。
4. 新结果覆盖旧的 AI 总览，历史结果可在操作审计中记录。

## 7. 流水线任务配置变化

当前代码评审任务配置保留：

| 字段 | 说明 |
| --- | --- |
| 任务名称 | 代码评审任务名称 |
| 源分支 | 参与对比的源分支 |
| 目标分支 | 参与对比的目标分支 |
| 评审人 | 人工审批人 |
| 通过方式 | 或 / 与 |

新增「AI 辅助评审」配置块。一期仅提供开关：

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 启用 AI 辅助评审 | 开关 | 关闭 | 启用后进入代码评审任务时触发 AI 初审 |

### 7.1 推荐默认值

第一期建议默认：

1. 新建代码评审任务时，AI 辅助评审默认关闭。
2. 启用后，系统在流水线进入代码评审任务时自动评审。
3. 本期不提供触发方式、评审范围、评审规则、阻断策略配置。

## 8. 评审列表页变化

现有列表字段包括流水线执行 ID、流水线、所属应用、代码仓库、代码项目、源分支、目标分支、触发用户、审核方式、提交时间、状态、操作。

新增字段：

| 字段 | 说明 |
| --- | --- |
| AI 评审 | 展示未启用、待评审、评审中、无明显风险、有风险、评审失败 |
| 风险等级 | 展示最高风险等级：严重、高、中、低、无 |
| 问题数 | 展示 AI 发现的问题总数和分级数量 |
| AI 摘要 | 一句话描述本次变更主要风险 |

### 8.1 列表筛选

新增筛选：

1. AI 评审状态：全部、未启用、待评审、评审中、无明显风险、有风险、评审失败。
2. 风险等级：全部、严重、高、中、低、无。
3. 是否阻断：全部、已阻断、未阻断。

## 9. 评审详情页变化

### 9.1 顶部信息区

在现有审核状态旁增加 AI 状态：

```text
审核状态：待审核
AI 评审：中风险 2个问题
```

AI 状态枚举：

| 状态 | 展示文案 |
| --- | --- |
| DISABLED | 未启用 |
| PENDING | 待评审 |
| RUNNING | 评审中 |
| PASSED | 无明显风险 |
| RISKY | 有风险 |
| FAILED | 评审失败 |

### 9.2 Diff 工具栏

在 Files Changed、Lines Changed、Commits、Theme、评审按钮附近增加：

1. AI 评审结果标签。
2. 重新评审按钮。
3. AI 评审抽屉入口。

示例：

```text
Files Changed: 1  Lines Changed: +8 -0  Commits: 3  AI 评审：中风险 2  重新评审  评审
```

### 9.3 文件目录

文件目录增加风险标识：

```text
JobInstanceServiceImpl.java  中 2
```

点击文件风险标识后，跳转到该文件的第一个 AI 评论位置。

### 9.4 行内 AI 评论

AI 评论展示在 diff 对应代码行下方，内容包括：

1. 风险等级。
2. 问题标题。
3. 问题说明。
4. 修复建议。
5. 反馈动作。

反馈动作：

| 动作 | 说明 |
| --- | --- |
| 采纳 | 用户认可该建议，并计划处理 |
| 忽略 | 用户认为不需要处理 |
| 误报 | 用户认为 AI 判断错误 |
| 转人工确认 | 将问题保留给人工评审人确认 |

### 9.5 AI 评审抽屉

右侧抽屉包含：

1. 评审结论。
2. 变更摘要。
3. 发现问题列表。
4. 测试建议。
5. 阻断说明。
6. 重新评审入口。

## 10. 审批规则

人工审批状态与 AI 评审状态必须分离：

| 类型 | 状态来源 | 作用 |
| --- | --- | --- |
| 人工审批状态 | 评审人操作 | 决定评审任务是否通过 |
| AI 评审状态 | AI 自动评审任务 | 提供风险提示和评审建议 |

### 10.1 一期审批规则

1. AI 评审结果不直接改变人工审批状态。
2. AI 评审结果不阻断流水线通过。
3. 评审人点击通过或驳回时，仍按现有「或 / 与」通过方式判断最终状态。
4. 如果 AI 发现风险，评审弹窗可提示风险数量，但不限制评审人继续操作。

评审弹窗提示示例：

```text
AI 评审发现 2 个问题，其中 1 个中风险。
AI 结果仅供参考，是否继续通过？
```

## 12. 接口建议

### 12.1 查询评审列表

列表接口新增返回：

```json
{
  "aiReviewStatus": "RISKY",
  "aiRiskLevel": "MEDIUM",
  "aiIssueCount": 2,
  "aiSummary": "本次变更直接修改入参对象，建议关注副作用风险"
}
```

### 12.2 查询 AI 评审详情

```http
GET /api/devops/code-reviews/{reviewId}/ai-review
```

返回 AI 摘要、问题列表、测试建议和阻断状态。

### 12.3 重新触发 AI 评审

```http
POST /api/devops/code-reviews/{reviewId}/ai-review/retry
```

### 12.4 提交问题反馈

```http
POST /api/devops/code-reviews/{reviewId}/ai-review/issues/{issueId}/feedback
```

请求体：

```json
{
  "feedback": "FALSE_POSITIVE",
  "comment": "该对象后续不会复用"
}
```

## 13. 权限规则

| 操作 | 权限 |
| --- | --- |
| 查看 AI 评审结果 | 有代码评审查看权限 |
| 重新评审 | 评审人、提交人、流水线管理员 |
| 修改 AI 配置 | 流水线编辑权限 |

## 14. 异常处理

| 场景 | 处理 |
| --- | --- |
| AI 服务超时 | 展示评审失败，允许手动重试 |
| diff 过大 | 展示超出 AI 评审处理范围，允许人工继续评审 |
| 文件类型不支持 | 跳过该文件并展示说明 |
| 模型返回不稳定 | 保留原始错误码，提示重新评审 |
| AI 评审中源分支更新 | 标记结果可能过期，提示重新评审 |

## 15. 验收标准

1. 启用 AI 辅助评审后，流水线进入代码评审任务时可自动生成 AI 评审结果。
2. 评审列表可展示 AI 状态、风险等级、问题数和 AI 摘要。
3. 评审详情页可展示 AI 总览、右侧抽屉、文件风险标识和行内评论。
4. 用户可对 AI 问题执行采纳、忽略、误报、转人工确认。
5. 用户可手动重新触发 AI 评审。
6. 人工审批状态与 AI 评审状态分离展示。
7. AI 评审结果不影响现有人工审批通过规则。
8. AI 服务失败时不影响用户查看 diff 和执行人工评审。

## 16. 原型文件

原型文件：`ai-code-review-prototype.html`

原型包含三个核心页面：

1. 流水线任务配置：展示 AI 辅助评审配置块。
2. 代码评审列表：展示 AI 状态、风险等级、问题数和摘要。
3. 代码评审详情：展示 diff、AI 总览、右侧抽屉和行内 AI 评论。
