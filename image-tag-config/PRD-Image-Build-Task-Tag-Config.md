# 镜像构建任务支持独立镜像 Tag PRD

## 1. 需求背景

当前 Flux DevOps 的镜像构建任务主要按系统固定规则生成镜像后缀，流程配置中也支持通过变量影响镜像后缀。该方式可以满足基础构建场景，但配置粒度偏全局，缺少面向单个镜像构建任务的独立 Tag 配置能力。

当一条流水线包含多个镜像构建任务时，多个镜像往往需要按服务名、模块名或部署场景生成不同 Tag。现有固定规则和流程级变量不够灵活，容易导致多个镜像产物使用相同或难以区分的后缀。

典型问题：

1. 一条流水线同时构建多个镜像时，镜像 Tag 主要受固定规则或流程级变量影响，难以按任务分别配置。
2. 不同镜像无法按服务名、模块名或部署场景配置不同 Tag。
3. 后续部署、回滚、排障时，难以从镜像 Tag 判断镜像来源和版本。
4. 用户需要通过构建命令或变量组合绕行处理，配置成本高且容易出错。

阿里云云效的处理方式是将镜像仓库、镜像 Tag、Dockerfile、ContextPath 等配置放在具体镜像构建步骤中。多个镜像通过多个镜像构建步骤分别配置和产出，而不是依赖一个全局镜像后缀。

## 2. 需求目标

在镜像构建类任务中支持配置任务级“镜像 Tag”，使同一条流水线中的不同镜像构建任务可以生成不同镜像 Tag。

目标包括：

1. 支持单个镜像构建任务配置独立镜像 Tag。
2. 保留流程级镜像后缀变量作为默认值，兼容已有流水线。
3. 支持固定值和变量表达式，满足版本追溯和多服务区分。
4. 运行时按照优先级计算最终镜像 Tag。
5. 在构建日志或运行详情中展示本任务实际使用的镜像 Tag。

## 3. 需求范围

### 3.1 本期包含

仅在“镜像构建”分类下的任务增加配置：

| 任务 | 是否增加 | 说明 |
| --- | --- | --- |
| 两个阶段构建 | 是 | 会构建并推送镜像 |
| Dockerfile 构建 | 是 | 会构建并推送镜像 |
| 通用构建 | 是 | 会构建并推送镜像 |

### 3.2 本期不包含

| 任务 | 不增加原因 |
| --- | --- |
| Maven 构建 | 非镜像构建任务 |
| Python/Go/C++ 单元测试 | 测试任务，不产出镜像 |
| Jacoco 覆盖率采集 | 覆盖率任务，不产出镜像 |
| 交付大数据制品、交付实时计算制品、交付对象存储、交付 UDF | 交付制品任务，不定义镜像 Tag |
| DomeOS 部署升级、部署到主机 | 部署消费镜像，不负责定义镜像 Tag |

## 4. 页面设计

### 4.1 入口页面

页面入口：

```text
流水线详情 - 流程详情 - 编辑任务抽屉
```

适用任务：

```text
两个阶段构建
Dockerfile 构建
通用构建
```

### 4.2 字段名称

字段名称：

```text
镜像 Tag
```

说明：

1. 镜像 Tag 是 Docker 镜像版本标识，会拼接在镜像名称后。
2. 示例：`backend:CI_COMMIT_ID` 中，`CI_COMMIT_ID` 即为镜像 Tag。
3. 不建议字段名使用“镜像后缀 / Tag”，避免用户理解成普通字符串后缀。

### 4.3 字段位置

不同任务现有表单结构不同，字段位置按任务分别处理。

#### 4.3.1 通用构建

通用构建已有“镜像”字段，`镜像 Tag` 放在“镜像”字段之后、“构建命令”之前。

```text
镜像仓库类型
DomeOS 项目
DomeOS 工程
镜像
镜像 Tag
构建命令
环境变量
启动命令
工作路径
任务输出
```

#### 4.3.2 Dockerfile 构建

Dockerfile 构建没有单独“镜像”选择项，镜像归属由 DomeOS 项目/工程决定，因此 `镜像 Tag` 放在“DomeOS 工程”之后、“配置方式”之前。

```text
镜像仓库类型
DomeOS 项目
DomeOS 工程
镜像 Tag
配置方式
代码项目
Dockerfile 名称
构建路径
任务输出
```

#### 4.3.3 两个阶段构建

两个阶段构建没有独立“镜像”字段，`镜像 Tag` 放在“DomeOS 工程”之后、“编译阶段”之前。

```text
镜像仓库类型
DomeOS 项目
DomeOS 工程
镜像 Tag
编译阶段
运行阶段
任务输出
```

### 4.4 字段定义

| 字段 | 说明 |
| --- | --- |
| 字段名称 | 镜像 Tag |
| 控件类型 | 可输入下拉框 |
| 是否必填 | 否 |
| 默认值 | 空 |
| 占位提示 | 请选择镜像 Tag |
| 下拉选项 | 流程配置页面维护的全局变量 |
| 支持内容 | 从下拉选项选择变量、手动输入固定值、变量表达式、固定值与变量组合 |

默认值规则：

1. 输入框默认不预填值。
2. 仅展示 placeholder：`请选择镜像 Tag`。
3. 下拉选项展示流程配置页面维护的全局变量，仅展示变量名。
4. 用户可以从下拉框选择已有变量，也可以直接输入新的固定值或变量表达式。
5. 用户主动填写后，才作为当前任务的任务级镜像 Tag 生效。

### 4.5 问号提示

`镜像 Tag` 标题右侧展示问号提示。鼠标 hover 后展示：

```text
镜像 Tag 是镜像版本标识，会拼接在镜像名称后，例如 backend:CI_COMMIT_ID。
支持固定值或流程配置中维护的流水线变量。

不填写时使用默认镜像后缀。
```

说明：

1. 不在输入框下方展示额外说明文案。
2. 不增加“插入变量”按钮。
3. 变量说明统一放在问号 hover 中，保持表单简洁。
4. 下拉框只承载流程配置页面已维护的全局变量，选项仅展示变量名；用户仍可手动输入未出现在下拉选项中的新值。

## 5. 生效规则

任务级镜像 Tag 优先级高于流程级镜像后缀。

```text
任务级镜像 Tag > 流程级镜像后缀变量 > 系统默认后缀
```

规则说明：

1. 如果任务级“镜像 Tag”有值，则当前任务构建出的镜像使用任务级配置。
2. 如果任务级“镜像 Tag”为空，则继续使用流程配置中的镜像后缀变量。
3. 如果任务级“镜像 Tag”为空且流程级变量也为空，则使用系统当前默认后缀生成逻辑。
4. 历史流水线不受影响，未配置任务级镜像 Tag 的任务保持现有行为。

## 6. 示例

同一条流水线中，多个镜像构建任务可以分别配置不同镜像 Tag。

| 构建任务 | 镜像 Tag |
| --- | --- |
| 通用构建-backend | backend-CI_COMMIT_ID |
| Dockerfile 构建-frontend | frontend-CI_COMMIT_ID |
| 两个阶段构建-worker | worker-BUILD_NUMBER |

最终镜像示例：

```text
backend:backend-6fbf207a
frontend:frontend-6fbf207a
worker:worker-1024
```

如果三个任务都不填写“镜像 Tag”，则三个任务均沿用流程配置中的默认镜像后缀，保持现有逻辑。

## 7. 变量支持

本期字段支持引用现有流水线变量，至少包括：

```text
CI_COMMIT_ID
CI_COMMIT_SHA
CI_COMMIT_REF_NAME
BUILD_NUMBER
```

说明：

1. 三类镜像构建任务复用同一套流水线变量体系。
2. 每个任务的“镜像 Tag”配置彼此独立。
3. 不同任务可以配置相同表达式，也可以配置不同表达式。

## 8. 运行展示

为便于排障和追溯，构建运行时需要展示最终生效的镜像 Tag。

展示位置：

1. 构建任务日志中输出最终镜像地址。
2. 最近运行/运行详情中展示当前任务产出的镜像信息。
3. 后续部署任务引用镜像时，优先展示任务输出中的最终镜像地址。

日志示例：

```text
镜像仓库：domeos.xxx.com/project/backend
镜像 Tag：backend-6fbf207a
最终镜像：domeos.xxx.com/project/backend:backend-6fbf207a
```

## 9. 兼容性

1. 已有流水线不自动填充任务级镜像 Tag。
2. 历史任务继续使用流程级镜像后缀变量。
3. 保存任务时，如果任务级镜像 Tag 为空，不改变现有构建行为。
4. API 需要允许任务配置中新增可选字段，例如 `imageTag`。

## 10. 异常与校验

建议校验：

1. 不允许输入空格。
2. 不允许输入镜像 Tag 不支持的特殊字符。
3. 长度建议不超过 128 个字符。
4. 变量表达式不存在时，保存可提示或运行时报错，具体沿用现有变量校验机制。

错误提示示例：

```text
镜像 Tag 只能包含字母、数字、下划线、点、短横线和变量表达式。
```

## 11. 验收标准

1. 新建或编辑“两个阶段构建 / Dockerfile 构建 / 通用构建”任务时，可以看到“镜像 Tag”字段。
2. “镜像 Tag”字段默认为空，placeholder 为“请选择镜像 Tag”。
3. 通用构建中，“镜像 Tag”位于“镜像”字段之后、“构建命令”之前。
4. Dockerfile 构建中，“镜像 Tag”位于“DomeOS 工程”之后、“配置方式”之前。
5. 两个阶段构建中，“镜像 Tag”位于“DomeOS 工程”之后、“编译阶段”之前。
6. 点击“镜像 Tag”下拉框时，下拉选项展示流程配置页面维护的全局变量，仅展示变量名。
7. 用户可以选择下拉选项，也可以直接输入新的固定值或变量表达式。
8. “镜像 Tag”右侧问号 hover 后展示变量说明和继承规则。
9. 字段为空时，构建结果保持现有逻辑，使用流程级镜像后缀变量。
10. 字段有值时，当前任务构建出的镜像使用任务级镜像 Tag。
11. 同一条流水线中多个镜像构建任务可以配置不同镜像 Tag。
12. 非镜像构建类任务不展示该字段。
13. 历史流水线不受影响，无需用户重新配置。
14. 运行日志或运行详情中能看到当前任务实际使用的镜像 Tag。
